# NaOCl Storage v7

Έκδοση χωρίς «χωρητικότητα στόχου».

Η κάρτα σύνοψης **Σύνολο ποσότητας δεξαμενών** δείχνει μόνο το άθροισμα των πραγματικών ποσοτήτων που δηλώνεις σε κάθε δεξαμενή:

`Πραγματικά m × tn/m` για κάθε δεξαμενή και μετά άθροισμα όλων.

Ανέβασε όλα τα αρχεία στο GitHub Pages. Αν βλέπεις παλιά τιμή, κάνε refresh ή διέγραψε προσωρινά τα δεδομένα από το κουμπί Reset.

## v8 changes
- Προστέθηκαν κουμπιά +1 βυτίο και +2 βυτία ανά δεξαμενή.
- Κάθε βυτίο υπολογίζεται ως 24.5 tn και αφαιρείται από την ποσότητα που λείπει μέχρι τον στόχο.
- Προστέθηκε πεδίο «Στόχοι m» ανά δεξαμενή. Παράδειγμα: `2, 6.20` σημαίνει πρώτα γέμισμα μέχρι 2m και μετά μέχρι 6.20m.


## v8.1 changes
- Το πεδίο «Σειρά» δέχεται πολλαπλές εμφανίσεις για την ίδια δεξαμενή, π.χ. `1-5`, `1,5` ή `1 5`.
- Στην πρώτη εμφάνιση χρησιμοποιείται η στάθμη που έχει δηλωθεί τώρα στη δεξαμενή.
- Σε κάθε επόμενη εμφάνιση της ίδιας δεξαμενής ο υπολογισμός ξεκινά από `0 m`, σαν να έχει αδειάσει και να ξαναγεμίζει.
- Το πρόγραμμα εμφανίζει «από Xm μέχρι Ym» και σημείωση «ξανά από 0» στις επαναλήψεις.


## v8.2 changes
- Προστέθηκαν ξεχωριστά πεδία ανά δεξαμενή για `1η σειρά`, `1ος στόχος m`, `2η σειρά`, `2ος στόχος m`.
- Παράδειγμα D1: 1η σειρά `1` με στόχο `2,0 m` και 2η σειρά `5` με στόχο `6,2 m`.
- Η πρώτη πλήρωση ξεκινά από την τωρινή δηλωμένη/υπολογιστική στάθμη.
- Η δεύτερη πλήρωση της ίδιας δεξαμενής ξεκινά από `0 m`, ώστε να υπολογίζεται σαν να άδειασε και ξαναγεμίζει.
- Η παλιά λειτουργία `Σειρά` και `Στόχοι m` παραμένει για συμβατότητα με ήδη αποθηκευμένα δεδομένα.


## Αλλαγή βυτίων
Τα +1/+2 βυτία υπολογίζονται μόνο στο Πρόγραμμα γεμίσματος ως ταυτόχρονη έξοδος από τη δεξαμενή όσο αυτή συνεχίζει να γεμίζει. Δεν αλλάζουν τη στάθμη/απεικόνιση της κάρτας. Ο χρόνος προκύπτει από: ποσότητα μέχρι στόχο + βυτία × 24,5 tn.
